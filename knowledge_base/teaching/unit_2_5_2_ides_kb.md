# Unit 2.5.2: Integrated Development Environments (IDEs)
## OCR J277 GCSE Computer Science -- Knowledge Base File

---

## Specification Requirements

**Students must know:**
- An IDE is a software application that provides tools to help programmers write, test, and debug programs
- **Four tools/facilities required:**

| IDE Tool | What it Does |
|----------|-------------|
| **Editor** | Where code is written; includes syntax highlighting (colour-coding keywords), auto-indentation, and auto-completion |
| **Error diagnostics** | Detects and reports errors; highlights syntax errors as the programmer types; shows error messages indicating type and location |
| **Run-time environment** | Allows the programmer to execute the program within the IDE; provides a console for input/output |
| **Translator** | The built-in compiler or interpreter that translates the code |

**Key distinctions:**
- The **editor** is one of four tools within the IDE -- the IDE is the complete application containing all four
- **Error diagnostics** is specific -- "it finds mistakes" is too vague; must say what it does (highlights, shows error messages, underlines problematic code)
- Students should have practical experience using an IDE

**Marking language:**
- Accept: "Where code is written" // "Syntax highlighting" // "Auto-indentation" // "Auto-completion" // "Keyword highlighting -- colours keywords/variables" // "Keyword completion -- suggests syntax when first part typed" for editor
- Accept: "Detects/highlights/reports errors" // "Shows error messages" // "Highlights syntax errors as programmer types" // "Shows location and detail of errors" // "Suggests fixes" for error diagnostics
- Accept: "Allows the programmer to run/execute the program within the IDE" // "Shows output of the program" for run-time environment
- Accept: "Translates the code" // "Built-in compiler or interpreter" // "Converts to machine code" for translator
- Accept: Other sensible IDE tools with matching descriptions (e.g. debugger, stepping, variable watch, breakpoints) -- examiners award generously where the description adds to the tool name. Source: June 2025 J277/02 Q2(c) MS
- Do not accept: "The IDE is where you type code" -- this describes the editor, not the IDE itself
- Do not accept: Vague descriptions for error diagnostics (e.g. "it finds mistakes") -- must say how
- Do not accept: Repetition as a description (e.g. "code editor -- edits code") -- description must add more than the tool name. Source: June 2025 J277/02 Q2(c) MS
- Do not accept: Features of programming languages (e.g. commenting, casting, variables) as IDE features. Source: June 2025 J277/02 ER Q2(c)

**Not required:** Specific IDE applications; debugger breakpoints or watch windows (accepted if offered)

## Examiner Intelligence

### Recurring Misconceptions (flagged in examiner reports 2022--2025)

1. **Confusing the IDE with the editor**
   - "The IDE is where you type code" -- this describes the editor, one tool inside the IDE. The IDE is the complete development environment containing all four tools.
   - Source: General ER trend 2022--2025

2. **Vague descriptions for error diagnostics**
   - "It finds mistakes" is insufficient. Must say how: highlights errors as typed, shows error messages, underlines problematic code, reports the type and location of the error.

3. **Students with less practical IDE experience answer poorly**
   - Candidates who had used an IDE throughout the course answered IDE questions more accurately.
   - Source: June 2023 J277/02 ER (general observation)

### Common Mark Losses
- Confusing the IDE with the editor
- "It finds mistakes" for error diagnostics without specifying how
- Describing only one tool when two or more are required

### Question Patterns (2022--2025)

| Year | Paper | Question | Topic Focus | Marks | Format |
|------|-------|----------|-------------|-------|--------|
| 2025 | J277/02 | Q4(a) | IDE tool -- identify from description | 2 | Identify |
| 2025 | J277/02 | Q4(b) | Describe how an IDE tool helps the programmer | 2 | Describe |

**Key observations:**
- IDE tools appeared explicitly in 2025 after being absent from 2022--2024; now active and likely to recur
- Identify-from-description and describe-how-it-helps are the established formats
- Practical experience with an IDE is the single best preparation for these questions

## Quick Reference

### IDE Tools at a Glance

| Tool | Purpose | Key detail |
|------|---------|------------|
| Editor | Write code | Syntax highlighting, auto-indentation, auto-completion |
| Error diagnostics | Detect and report errors | Highlights errors as typed; shows error messages |
| Run-time environment | Run the program within the IDE | Console for input/output during execution |
| Translator | Translate the code | Built-in compiler or interpreter |

### Key Exam Language

**Say this:**
- "Syntax highlighting colour-codes keywords as the programmer types" (editor)
- "Highlights syntax errors and shows error messages indicating type and location" (error diagnostics)
- "Allows the programmer to run the program within the IDE and see output" (run-time environment)
- "Translates the source code using a built-in compiler or interpreter" (translator)

**Not this:**
- "The IDE is where you type code" -- that is the editor
- "It finds mistakes" for error diagnostics -- must specify how
